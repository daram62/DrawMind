# Requirements Document

## Introduction

"The Forest Within (내 안의 숲)"은 사용자가 직접 그린 그림으로 감정 세계를 치유하는 AI 기반 인터랙티브 시네마틱 경험입니다. 사용자는 5개의 감정 챕터(불안, 후회, 분노, 자기의심, 회복)를 거치며 그림을 그리고, Gemini AI가 그림을 분석하여 감정을 해석하고 치유적 내러티브를 생성합니다. 최종적으로 모든 그림과 여정이 Sora2로 생성된 AI 쇼츠 영상으로 완성됩니다.

**핵심 경험:** "내가 그린 그림이 내면의 세계를 변화시킨다"

**UI 스타일:** 텔레스트레이션/갈틱폰 스타일의 귀여운 스케치북 느낌

## Requirements

### Requirement 1

**User Story:** 사용자로서, 귀여운 스케치북 스타일의 캔버스에 자유롭게 그림을 그릴 수 있어야 한다. 그래야 감정을 편안하게 표현할 수 있다.

#### Acceptance Criteria

1. WHEN 사용자가 챕터를 시작할 때 THEN 시스템은 스케치북 스타일의 캔버스를 제공해야 한다
2. WHEN 사용자가 그림을 그릴 때 THEN 시스템은 터치/마우스 입력을 부드럽게 지원해야 한다
3. WHEN 사용자가 도구를 선택할 때 THEN 시스템은 펜, 지우개, 색상 팔레트, 브러시 크기를 제공해야 한다
4. WHEN 사용자가 실수할 때 THEN 시스템은 Undo/Redo 기능을 제공해야 한다
5. WHEN 사용자가 그림을 완성할 때 THEN 시스템은 "다음으로" 버튼을 활성화해야 한다
6. IF 사용자가 캔버스를 비우고 싶을 때 THEN 시스템은 "전체 지우기" 기능을 제공해야 한다

### Requirement 2

**User Story:** 사용자로서, 5개의 감정 챕터를 순차적으로 경험하며 각 챕터의 주제에 맞는 그림을 그리고 싶다. 그래야 감정 여정을 체계적으로 탐색할 수 있다.

#### Acceptance Criteria

1. WHEN 사용자가 여정을 시작할 때 THEN 시스템은 프롤로그 화면을 보여줘야 한다
2. WHEN 프롤로그가 끝날 때 THEN 시스템은 1장(불안)부터 시작해야 한다
3. WHEN 각 챕터가 시작될 때 THEN 시스템은 감정 테마와 그림 프롬프트를 보여줘야 한다
4. WHEN 사용자가 그림을 완성할 때 THEN 시스템은 다음 챕터로 자동 전환해야 한다
5. WHEN 모든 챕터가 완료될 때 THEN 시스템은 에필로그로 이동해야 한다
6. IF 사용자가 중간에 나갈 때 THEN 시스템은 진행 상황을 저장해야 한다

### Requirement 3

**User Story:** 사용자로서, 내가 그린 그림이 Gemini AI에 의해 분석되고 감정적 피드백을 받고 싶다. 그래야 내 감정을 이해하고 치유할 수 있다.

#### Acceptance Criteria

1. WHEN 사용자가 그림을 제출할 때 THEN 시스템은 Gemini Vision API로 그림을 분석해야 한다
2. WHEN Gemini가 그림을 분석할 때 THEN 시스템은 색상, 형태, 감정 키워드를 추출해야 한다
3. WHEN 분석이 완료될 때 THEN 시스템은 감정 점수(불안도, 긍정도 등)를 계산해야 한다
4. WHEN 감정 분석 결과가 나올 때 THEN 시스템은 치유적 피드백 메시지를 생성해야 한다
5. WHEN 피드백이 생성될 때 THEN 시스템은 부드러운 애니메이션과 함께 표시해야 한다
6. IF 분석이 실패할 때 THEN 시스템은 재시도 옵션을 제공해야 한다

### Requirement 4

**User Story:** 사용자로서, AI가 생성한 치유적 내러티브와 동반자 캐릭터의 대화를 경험하고 싶다. 그래야 감정적 연결감을 느낄 수 있다.

#### Acceptance Criteria

1. WHEN 프롤로그에서 첫 그림을 그릴 때 THEN 시스템은 Gemini로 동반자 캐릭터를 생성해야 한다
2. WHEN 각 챕터가 진행될 때 THEN 시스템은 Gemini Pro로 스토리 내러티브를 생성해야 한다
3. WHEN 내러티브가 생성될 때 THEN 시스템은 동반자 캐릭터의 대화 형식으로 표시해야 한다
4. WHEN 대화가 표시될 때 THEN 시스템은 타이핑 애니메이션 효과를 적용해야 한다
5. WHEN 사용자가 내러티브를 읽을 때 THEN 시스템은 배경 음악과 효과음을 재생해야 한다
6. IF 내러티브가 길 때 THEN 시스템은 스크롤 가능한 대화창을 제공해야 한다

### Requirement 5

**User Story:** 사용자로서, 내 감정 여정의 변화를 시각적으로 확인하고 싶다. 그래야 치유 과정을 실감할 수 있다.

#### Acceptance Criteria

1. WHEN 사용자가 챕터를 완료할 때마다 THEN 시스템은 감정 데이터를 누적 저장해야 한다
2. WHEN 에필로그에 도달할 때 THEN 시스템은 감정 변화 그래프를 생성해야 한다
3. WHEN 그래프가 표시될 때 THEN 시스템은 불안→수용→회복의 곡선을 보여줘야 한다
4. WHEN 사용자가 그래프를 볼 때 THEN 시스템은 각 챕터의 그림 썸네일을 함께 표시해야 한다
5. WHEN 감정 여정이 완료될 때 THEN 시스템은 "당신의 숲이 회복되었습니다" 메시지를 보여줘야 한다

### Requirement 6

**User Story:** 사용자로서, 내 감정 여정이 담긴 AI 생성 쇼츠 영상을 받고 싶다. 그래야 나의 치유 과정을 기억하고 공유할 수 있다.

#### Acceptance Criteria

1. WHEN 에필로그가 시작될 때 THEN 시스템은 Sora2 API로 영상 생성을 요청해야 한다
2. WHEN 영상을 생성할 때 THEN 시스템은 모든 그림과 내러티브를 프롬프트로 전달해야 한다
3. WHEN 영상이 생성될 때 THEN 시스템은 9:16 세로 포맷의 쇼츠 영상을 생성해야 한다
4. WHEN 영상이 완성될 때 THEN 시스템은 다운로드 버튼을 제공해야 한다
5. WHEN 사용자가 영상을 다운로드할 때 THEN 시스템은 MP4 포맷으로 저장해야 한다
6. IF 영상 생성이 오래 걸릴 때 THEN 시스템은 진행 상황 표시줄을 보여줘야 한다

### Requirement 7

**User Story:** 사용자로서, 스케치북 스타일의 귀여운 UI에서 편안하게 여정을 경험하고 싶다. 그래야 치유적 분위기를 느낄 수 있다.

#### Acceptance Criteria

1. WHEN 사용자가 앱에 접속할 때 THEN 시스템은 스케치북 테마의 배경을 표시해야 한다
2. WHEN 캔버스가 표시될 때 THEN 시스템은 종이 질감과 테두리를 적용해야 한다
3. WHEN 버튼이 표시될 때 THEN 시스템은 손그림 스타일의 아이콘을 사용해야 한다
4. WHEN 챕터가 전환될 때 THEN 시스템은 페이지 넘김 애니메이션을 적용해야 한다
5. WHEN 색상 팔레트가 표시될 때 THEN 시스템은 크레용/색연필 스타일을 적용해야 한다
6. WHEN 로딩 중일 때 THEN 시스템은 귀여운 로딩 애니메이션(나뭇잎, 별 등)을 보여줘야 한다

### Requirement 8

**User Story:** 사용자로서, 내 여정 데이터가 안전하게 저장되고 나중에 다시 볼 수 있어야 한다. 그래야 치유 과정을 돌아볼 수 있다.

#### Acceptance Criteria

1. WHEN 사용자가 그림을 그릴 때 THEN 시스템은 이미지를 S3에 저장해야 한다
2. WHEN 감정 분석이 완료될 때 THEN 시스템은 결과를 데이터베이스에 저장해야 한다
3. WHEN 여정이 완료될 때 THEN 시스템은 전체 세션 데이터를 저장해야 한다
4. WHEN 사용자가 재방문할 때 THEN 시스템은 이전 여정 목록을 보여줘야 한다
5. WHEN 사용자가 이전 여정을 선택할 때 THEN 시스템은 모든 그림과 내러티브를 다시 보여줘야 한다
6. IF 사용자가 여정을 삭제하고 싶을 때 THEN 시스템은 확인 후 삭제 기능을 제공해야 한다

### Requirement 9

**User Story:** 개발자로서, 챕터별 프롬프트와 감정 분석 로직을 쉽게 관리하고 수정할 수 있어야 한다. 그래야 해커톤 중 빠르게 조정할 수 있다.

#### Acceptance Criteria

1. WHEN 챕터 데이터를 정의할 때 THEN 시스템은 JSON 설정 파일을 사용해야 한다
2. WHEN Gemini 프롬프트를 작성할 때 THEN 시스템은 템플릿 시스템을 사용해야 한다
3. WHEN 감정 키워드를 매핑할 때 THEN 시스템은 설정 파일에서 관리해야 한다
4. WHEN 내러티브 톤을 조정할 때 THEN 시스템은 환경 변수로 제어할 수 있어야 한다
5. WHEN 새로운 챕터를 추가할 때 THEN 시스템은 코드 수정 없이 설정만으로 가능해야 한다

### Requirement 10

**User Story:** 사용자로서, 모바일과 데스크톱 모두에서 편안하게 그림을 그리고 싶다. 그래야 어디서든 여정을 경험할 수 있다.

#### Acceptance Criteria

1. WHEN 모바일에서 접속할 때 THEN 시스템은 터치 입력을 최적화해야 한다
2. WHEN 데스크톱에서 접속할 때 THEN 시스템은 마우스 입력을 최적화해야 한다
3. WHEN 화면 크기가 변할 때 THEN 시스템은 캔버스 크기를 자동 조정해야 한다
4. WHEN 세로 모드일 때 THEN 시스템은 UI를 세로 레이아웃으로 표시해야 한다
5. WHEN 가로 모드일 때 THEN 시스템은 UI를 가로 레이아웃으로 표시해야 한다
6. IF 화면이 작을 때 THEN 시스템은 도구 팔레트를 접을 수 있어야 한다
